* Tutorial

  Este  diretório  contém todos  os  fascículos  do  tutorial de  construção  de
  compiladores de  Jack Crenshaw,  incluindo o novo  Fascículo 15.   A audiência
  pretendida é  a daquelas  pessoas que  não são  cientistas de  computação, mas
  divertem-se  com  computação   e  sempre  tiveram  o  desejo   de  saber  como
  compiladores funcionam.   Um monte de  teoria dos compiladores foi  deixada de
  lado,  mas as  questões  práticas foram  cobertas.  No  momento  em que  tiver
  completado a  série, você  será capaz  de projetar e  construir o  seu próprio
  compilador que  funciona. Ele não será  o melhor do mundo,  nem emitirá código
  incrivelmente firme.   Seu produto não vai  tirar a Borland ou  a MicroSoft do
  mercado. Mas vai funcionar, e vai ser seu.

  Uma  palavra sobre  o  formato  de arquivo:  Os  arquivos foram  originalmente
  criados usando o editor DOS do  Borland, Sprint. Sprint podia escrever para um
  arquivo de texto somente se você formatasse o arquivo para ir até a impressora
  selecionada.  Eu  usei a  impressora mais  comum que  poderia pensar,  a Epson
  MX-80, mas  mesmo assim  os arquivos  acabavam com  sequências de  controle no
  início e no fim de cada página.

  Para  atualizar estes  arquivos e  me posicionar  para continuar  a série,  eu
  recentemente (1994) converti todos os  arquivos para trabalhar com o Microsoft
  Word para  Windows. Ao contrário  do Sprint, Word  permite que você  escreva o
  arquivo  como um  arquivo  de texto  DOS. Infelizmente,  isto  medava um  novo
  problema, porque quando  o Word está escrevendo para um  arquivo de texto, ele
  não  escreve  quebras de  página  rígidas  ou  números  de página.  Em  outras
  palavras, em seis anos fomos de um  arquivo com quebras de página e números de
  página  mas  com  sequências  de  controle  embutidas,  para  um  arquivo  sem
  sequências de  controle embutidas  mas sem  quebras de  página nem  números de
  página. O progresso não é maravilhoso?

  Claro, é  possível para  mim inserir  os números  como texto  puro, em  vez de
  delegar para  o editor que  os escreva para  mim. Mas dado  que o Word  não me
  permite inserir quebras  de página ao arquivo, nós acabaríamos  com números de
  página que  podem ou  não podem  cair no  fim das  páginas, dependendo  de seu
  editor e de sua impressora. Me parece  que quase todo arquivo que já baixei da
  CompuServe ou dos  BBS's que tinha tais numerações de  página era incompatível
  com minha impressora, e me dava páginas  que eram uma linha mais curta ou mais
  longa, com os números de página consequentemente deslocando-se pela página.

  Então  talvez este  novo  formato é,  afinal  de contas,  o  mais seguro  para
  distribuição  geral.  Os  arquivos  como estão  aparecerão  sem  problemas  em
  qualquer editor  de textos  capaz de  ler arquivos  de texto  DOS. Dado  que a
  maioria  dos  editores  de  texto  de  hoje  em  dua  incluem  ferramentas  de
  processamento bastante sofisticadas, você deve conseguir que seu editor pagine
  para você antes de mandar para a impressora.

  Espero que  goste dos  tutoriais. Muito  foi pensado para  que eles  viessem à
  tona.
